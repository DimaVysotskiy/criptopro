# Настройка КриптоПро CSP для работы с ЭП на ОС linux

---

## Вступительная часть 

В руководстве рассмотрены два подхода к работе с КриптоПро:
- Первый — аналогичный работе в Windows: ключевые контейнеры создаются автоматически, а информация сертификата фиксируется при регистрации и не изменяется.
- Второй — работа через терминал Linux, где все действия выполняются вручную, что даёт полный контроль над процессом.

--- 

## Установка КриптоПро

Ссылка на инструкцию установки: https://www.altlinux.org/КриптоПро#Установка_КриптоПро_CSP

> **Примечание.** Для удобства пользования КриптоПро добавьте каталоги в переменную окружения `PATH` в файле `~/.bashrc`:
> ```bash
> export PATH="/opt/cprocsp/bin/amd64:/opt/cprocsp/sbin/amd64:$PATH"
> ```

> **Примечание.** GUI «Инструменты Крипто Про» отстаёт от терминала. Перезапускайте GUI для отображения актуальной информации.

### Основные инструменты

| Утилита | Описание |
|---------|----------|
| `cpconfig` | Настройка криптопровайдера |
| `cryptcp` | Подпись и шифрование файлов |
| `certmgr` | Менеджер сертификатов |
| `csptest` | Тестирование и управление контейнерами |
| `cpverify` | Проверка подписи |

### Каталог хранения ключей, пользователей и прочих данных:
```
/var/opt/cprocsp/
```

Чтобы система знала, сертификатам какого УЦ доверять.

**Ссылки для скачивания сертификатов Тестового УЦ КриптоПро:**
- Корневой сертификат: http://testca2012.cryptopro.ru/cert/rootca.cer
- Промежуточный сертификат: http://testca2012.cryptopro.ru/cert/subca.cer


**Далее эти сертификаты нужно установить в Крипто Про:**

3) Можно проверить, что сертификаты установились в хранилища.

## Способ №2 (делаем всё через терминал):

### Этап 1. Установка корневого Тестового УЦ Крипто Про (промежуточного сертификата нету и нету даже в цепочке сертификатов).

#### 1. Нужно скачать корневой сертификат УЦ с сайта:

**Зачем нужны сертификаты УЦ?**
Чтобы система знала, сертификатам какого УЦ доверять.
![alt text](image.png)

**Ссылки для скачивания сертификатов Тестового УЦ КриптоПро (выбирать Base64):**
https://testgost2012.cryptopro.ru/certsrv/certcarc.asp

#### 2. Далее надо этот сертификат установить в хранилище:

**Команды установки сертификата**

Установка для текущего пользователя:
```bash
certmgr -install -store u<название_хранилища> -file <путь_к_сертификату>
```

Установка на компьютер:
```bash
certmgr -install -store m<название_хранилища> -file <путь_к_сертификату>
```

##### Хранилища сертификатов

| Английское имя | Куда можно установить | Русское название в интерфейсе |
|---|---|---|
| `my` | Default | Личное |
| `root` | Default / LocalMachine | Доверенные корневые центры сертификации |
| `ca` | Default / LocalMachine | Промежуточные центры сертификации |
| `addressbook` | Default / LocalMachine | Другие пользователи |
| `cache` | Default | Кэш |
| `request` | Default | Запросы заявок на сертификат |
| `cryptoprotrustedstore` | Default / LocalMachine | Доверенные корневые сертификаты КриптоПро CSP |
| `disallowed` | Default / LocalMachine | Недоверенные сертификаты *(обычно скрыто, но существует)* |

### Этап 2. Создание ключевого контейнера.

Для корректной работы с ЭП нужно иметь закрытый и открытый ключ — они хранятся в ключевом контейнере. Создадим ключевой контейнер.

Команда создания ключевого контейнера:
```bash
csptest -keyset -newkeyset -container '\\.\HDIMAGE\<имя_контейнера>'
```

> **Примечание 1.** Имя контейнера состоит из \\\\.\\<имя считывателя>\\<имя_контейнера>. Чтобы посмотреть доступные считыватели и их имена воспользуйтесь командой: 
> ```bash
> cpconfig -hardware reader -view
> ```
> В гайде поумолчанию используется считыватель с ПК: HDIMAGE


> **Примечание 2.** По умолчанию тип криптопровайдера при создании ключевой пары — `Crypto-Pro GOST R 34.10-2012 Cryptographic Service Provider`, поэтому параметр `-provtype 80` в команде не обязателен.

> **Примечание 3.** По умолчанию в ключевой контейнер устанавливаются два секретных ключа: `signature` и `exchange`. Чтобы изменить это поведение, используйте `-keytype <type>`.

#### Виды ключей (общая информация):

| Ключ | Назначение |
|------|------------|
| `signature key` | Создание электронной подписи; подтверждает авторство и целостность документа |
| `exchange key` | Шифрование/расшифровка данных при обмене информацией с другими пользователями |
| `symmetric key` | Быстрое шифрование больших объёмов данных между доверенными сторонами с одним ключом |

### Этап 3. Создание запроса на выпуск сертификата от УЦ.

Чтобы ЭП считалась квалифицированной, необходимо получить публичный сертификат от УЦ. Для этого создаётся заявка (CSR).

**Команда создания заявки:**
```bash
cryptcp -creatrqst -dn 'CN=test_user,E=test_user@altlinux.org' -nokeygen -cont '\\.\HDIMAGE\<имя_контейнера>' <имя_файла_заявки>.csr -base64 -certusage "1.3.6.1.5.5.7.3.3,1.3.6.1.4.1.311.10.3.12,1.3.6.1.5.5.7.3.4,1.3.6.1.4.1.311.10.5.1"
```

> **Примечание 1.** Параметр `-base64` обязателен, чтобы не пришлось вручную переводить файл `.csr` в читаемый формат. Строка -dn является примером.

> **Примечание 2.** Параметр `-nokeygen` обязателен, чтобы заявка создавалась по уже созданому ключевому контейнейру, а не создавала новый. Мы создавали контейнер сами, чтобы он был с нужными нам секретными ключами, если убрать параметр `-nokeygen`, то автоматически созданный ключевой контейнер может не иметь нужных нам секретных ключей. 

#### Назначение сертификата

> **Примечание.** С помощью опции `-certusage` можно указать OID назначение сертификата. В таблице ниже приведены доступные типы применения:

### Этап 4. Получение публичного сертификата от УЦ.

1. Открыть файл `<имя_файла_заявки>.csr` в любом текстовом редакторе (я использую nano).
2. Удалить строку `-----BEGIN CERTIFICATE-----` в начале и `-----END CERTIFICATE-----` в конце.
3. Скопировать оставшееся содержимое.
4. Перейти по ссылке https://testgost2012.cryptopro.ru/certsrv/certrqxt.asp и вставить содержимое буфера в поле «Сохранённый запрос:».
5. Нажать кнопку **«Выдать»**.
6. Выбрать формат **Base-64** и нажать **«Загрузить сертификат»**.

**Сертификат получен!!!**

---

### Этап 5. Добавление сертификата от УЦ

Полученный от УЦ сертификат нужно поместить в ключевой контейнер, а затем добавить в Личные сертификаты.

**Добавление сертификата в контейнер**:
```bash
cryptcp -instcert -cont '\\.\HDIMAGE\<имя_контейнера>' -ku <путь до>/<сертификата>.cer
```

> **Примечание.** Параметр `-ku` означает, что сертификат устанавливается в контейнер пользователя.

**Добавление сертификата в Личные**:
```bash
certmgr -inst -store uMy -file <имя_сертификата>.cer
```
